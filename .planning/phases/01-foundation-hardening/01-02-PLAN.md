**Per D-13, D-14, D-15, D-16:** Create the centralized formatting utility using `Intl.NumberFormat`.

1. **Create `frontend/src/types/formatting.ts`:**
   ```typescript
   export type FormatType = 'number' | 'currency' | 'percentage' | 'decimal'

   export interface FormatNumberOptions {
     type: FormatType
     decimals?: number
     abbreviate?: boolean
     currencyCode?: string  // ISO 4217 code from companion column per D-16
   }
   ```

2. **Create `frontend/src/lib/formatters.ts`** (per D-13, the centralized utility):
   - Export `formatValue(value: number | null | undefined, options: FormatNumberOptions): string`
   - Export `formatValueFull(value: number | null | undefined, options: FormatNumberOptions): string`
   - If `value` is null/undefined, return `''`
   - For `type: 'currency'`:
     - If `currencyCode` is provided and valid, use `Intl.NumberFormat` with `{ style: 'currency', currency: currencyCode, minimumFractionDigits: decimals ?? 2, maximumFractionDigits: decimals ?? 2, notation: abbreviate ? 'compact' : 'standard' }`
     - If `currencyCode` is missing/undefined, fall back to plain number format (per D-14, no hardcoded defaults): `Intl.NumberFormat` with `{ minimumFractionDigits: decimals ?? 2, maximumFractionDigits: decimals ?? 2, notation: abbreviate ? 'compact' : 'standard' }`
   - For `type: 'percentage'`:
     - Use `Intl.NumberFormat` with `{ style: 'percent', minimumFractionDigits: decimals ?? 1, maximumFractionDigits: decimals ?? 1 }`
     - IMPORTANT: `Intl.NumberFormat` percent style expects the value as a fraction (0.55 for 55%), so divide `value` by 100 before formatting
   - For `type: 'number'`:
     - Use `Intl.NumberFormat` with `{ minimumFractionDigits: 0, maximumFractionDigits: decimals ?? 0, notation: abbreviate ? 'compact' : 'standard' }`
   - For `type: 'decimal'`:
     - Use `Intl.NumberFormat` with `{ minimumFractionDigits: decimals ?? 2, maximumFractionDigits: decimals ?? 2, notation: abbreviate ? 'compact' : 'standard' }`
   - `formatValueFull` calls `formatValue` with `{ ...options, abbreviate: false }`

3. **Update `frontend/src/components/shared/count-animation.tsx`:**
   - Remove the internal `formatNumber` function entirely
   - Import `formatValue` from `@/lib/formatters` and `FormatNumberOptions` from `@/types/formatting`
   - Update `CountAnimationProps` to use `FormatNumberOptions` fields: replace `format` prop with `formatOptions: FormatNumberOptions`
   - In the `useTransform` callback, call `formatValue(v, formatOptions)` instead of the old `formatNumber`
   - Keep `prefix` and `suffix` props for backward compatibility (they wrap the formatted output)
   - Per D-15: When `formatOptions.abbreviate` is true, render abbreviated number. The hover full number will be handled by the parent component (config-kpi-row), not here.

4. **Update `frontend/src/components/grid/cell-renderers/amount-cell.tsx`:**
   - Remove the inline `Intl.NumberFormat` call
   - Import `formatValue` from `@/lib/formatters`
   - Remove the hardcoded `'USD'` default (per D-14, no hardcoded defaults)
   - Read currency from `data?.currency` — if undefined, format as plain number
   - Call `formatValue(Math.abs(num), { type: 'currency', currencyCode: currency, decimals: 2 })`
   - Handle negative values: prepend `-` if `num < 0`

   - Add a `title` attribute (native HTML tooltip) with `formatValueFull(value, { type: kpi.format })` so hovering shows the full unabbreviated number
