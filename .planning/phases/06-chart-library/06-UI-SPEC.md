## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline badge padding, chart type grid icon margins |
| sm | 8px (`gap-2`, `p-2`) | Toolbar button gaps, accordion step summary padding, combobox internal gaps |
| md | 16px (`gap-4`, `p-4`) | Chart type grid gaps, chart library card grid gaps, accordion content padding |
| lg | 24px (`gap-6`, `p-6`) | Page padding, builder panel gap (accordion left vs preview right), list page sections |
| xl | 32px (`gap-8`) | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Accordion step collapsed summary height: 44px (`h-11`) -- enough for single-line summary text with icon
- Chart type grid items: 80px x 72px (`w-20 h-18`) -- icon (32px) + label (12px) with 8px gap
- Live preview area minimum height: 320px (`min-h-[320px]`) -- enough for chart rendering at readable scale
- Chart library card thumbnail area: 120px (`h-[120px]`) -- static chart preview placeholder

**Source:** CLAUDE.md (page padding `p-6`, section gaps `gap-6`, grid gaps `gap-4`), Phase 5 UI-SPEC (list page patterns)

## Focal Points

### Chart Library List Page

**Focal point:** The chart card grid (or row list). The toolbar, page title, and card wrapper are supporting chrome. The user's eye lands on chart cards with their type icons and thumbnail areas making the library feel visual and scannable. This is achieved by:
- Page title and toolbar are visually subdued (`text-muted-foreground` description, minimal toolbar height)
- Cards have `bg-card` surface contrast against `bg-background` page
- Card hover lift (`hover:shadow-md hover:-translate-y-0.5`) provides interactive affordance
- Chart type icon on each card gives immediate visual identification
- "+ New Chart" button uses accent fill, drawing the eye as the primary action

**Empty state focal point:** The `Empty` component centered in the card area with the "Create Chart" CTA button in accent fill.

### Chart Builder Page

**Focal point:** The live preview area on the right. The accordion stepper on the left is the workflow, but the preview is where the user sees the result of their choices. This is achieved by:
- Preview area takes at least 50% of horizontal space (`flex-1 min-w-[50%]`)
- Accordion panel is fixed width (`w-[380px]`) -- tight enough for forms, leaving maximum preview space
- Preview container has `bg-card border rounded-lg` making it a distinct surface
- The preview updates live as the user changes settings, creating a feedback loop that draws attention
- Active accordion step has a left accent border, guiding the eye back to the current task

### Chart Detail Panel

**Focal point:** The live chart render at the top of the side panel. Below the chart, metadata is secondary information.

| `treemap` | 1+ dimension, 1 measure | "Requires at least 1 dimension and 1 measure" |

## Interaction Contract

### Chart Library List Page

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Load chart list | Page mount | Skeleton grid/rows, then charts appear | <500ms cached, <2s cold |
| Search charts | Type in search input | Client-side filter on name and description | 0ms (instant filter) |
| Filter by chart type | Select from chart type dropdown | Client-side filter by chart_type | 0ms (instant filter) |
| Filter by dataset | Select from dataset dropdown | Client-side filter by dataset_id | 0ms (instant filter) |
| Switch view mode | Click grid/list toggle | View switches between card grid and row list | <16ms |
| Click chart card/row | Click card or row | Side panel (Sheet) opens with live chart render and metadata | <500ms for chart data fetch |
| Click "+ New Chart" | Click button | Navigate to `/charts/new` | Standard route transition |
| Hover chart card | Mouse enters card | Card lifts with shadow (`hover:shadow-md hover:-translate-y-0.5`) | <16ms (CSS transition) |
| Click "Edit" in detail panel | Click button | Navigate to `/charts/{id}/edit` | Standard route transition |
| Click "Delete" in detail panel | Click button | Delete confirmation dialog opens | <16ms |

### Chart Builder Page

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Load builder (create) | Page mount at `/charts/new` | Empty accordion with Step 1 expanded, preview area shows empty state | <200ms |
| Load builder (edit) | Page mount at `/charts/:id/edit` | Accordion with all steps pre-populated and collapsed, preview shows live chart | <1s |
| Select dataset (Step 1) | Open combobox, search, click dataset | Dataset selected, column metadata shows in preview, Step 2 unlocks | <500ms for column metadata |
| Click "Preview Data" (Step 1) | Click button in preview area | Fetches sample rows from dataset SQL, shows in table below column metadata | Standard query latency |
| Select chart type (Step 2) | Click compatible chart type icon | Type selected with ring highlight, Step 3 unlocks with type-specific mapping fields | <16ms |
| Click dimmed chart type (Step 2) | Click incompatible type | Nothing happens, tooltip explains why on hover | 0ms |
| Map columns (Step 3) | Select from role-filtered dropdowns | Preview updates with live chart using mapped columns | <200ms chart re-render |
| Add metric (Step 3) | Click "+ Add Metric" | New metric dropdown appears | <16ms |
| Remove metric (Step 3) | Click X on metric badge | Metric removed, preview updates | <200ms chart re-render |
| Change aggregation (Step 3) | Select from aggregation dropdown | Preview updates | <200ms chart re-render |
| Toggle legend (Step 4) | Click Switch | Preview updates with/without legend | <200ms chart re-render |
| Change legend position (Step 4) | Select position | Preview updates legend position | <200ms chart re-render |
| Edit title (Step 4) | Type in input | Preview updates chart title (debounced 300ms) | 300ms debounce |

| Change chart type (Step 2 re-edit) | Select different type | Step 3 mapping fields change to match new type, Steps 3-5 reset | <16ms |

| Click "Save Chart" (Step 5) | Click button | Chart saves to RecViz DB, toast confirms, navigate to chart library list (create) or stay (edit) | <1s |
| Click "Save Chart" without name | Click button | Button disabled, name input shows validation error | 0ms (prevented) |

|-------------|---------|----------|
