    **ChartBuilder orchestrator** (`chart-builder.tsx`):

    Props interface `ChartBuilderProps`:
    - `mode: 'create' | 'edit'`
    - `initialChart?: RecvizChart` (for edit mode, pre-populates all steps)
    - `onPreviewChange: (state: BuilderPreviewState) => void` (updates preview panel)

    Internal state:
    - `activeStep: string` -- controlled Accordion value, starts at `'dataset'`
    - `completedSteps: Set<string>` -- tracks which steps are done
    - Builder form state object:
      ```typescript
      interface BuilderState {
        datasetId: string | null
        dataset: RecvizDataset | null
        chartType: LibraryChartType | null
        columnMapping: ChartColumnMapping
        appearance: ChartAppearance
        name: string
        description: string
      }
      ```
    - Initialize from `initialChart` in edit mode (all steps pre-completed, all fields populated)

    Step management functions:
    - `completeStep(step: string, nextStep: string)`: adds step to `completedSteps`, sets `activeStep` to `nextStep`
    - `resetFromStep(step: string)`: removes all steps at or after the given step from `completedSteps`. The step order is `['dataset', 'type', 'mapping', 'appearance', 'save']`. When resetting from `'dataset'`, clear `chartType`, `columnMapping`, `appearance` fields. When resetting from `'type'`, clear `columnMapping` fields. Per D-02.
    - `isStepLocked(step: string)`: returns true if the previous step in the order is not in `completedSteps`. First step ('dataset') is never locked. Per D-02.

    Layout: Uses Shadcn `Accordion type="single" collapsible` with `value={activeStep}` and `onValueChange` that only allows expanding non-locked steps.

    Each `AccordionItem` has:
    - `value` = step key (e.g., `'dataset'`)
    - `disabled` = `isStepLocked(step)` 
    - Custom trigger showing: step number, step label, and when collapsed + completed: summary text in `text-sm text-muted-foreground`
    - Active step styling: `border-l-2 border-primary` on the left edge
    - Locked step styling: `opacity-50 cursor-not-allowed`
    - Content area: `p-4` padding

    Summary lines per step (per UI-SPEC Copywriting Contract):
    - Step 1: `"{dataset.name} . {dataset.columns.length} cols"` (e.g., "Sales Data . 12 cols")
    - Step 2: `CHART_DISPLAY_NAMES[chartType]` (e.g., "Bar Chart")
    - Step 3: `"{categoryColumn} x {metricColumns.join(', ')}"` (e.g., "Region x Amount, Count")
    - Step 4: `"Title: {title}"` or `"Default appearance"` if no title
    - Step 5: (no summary when save step, it's always the last)

    Save handler: Uses `useCreateChart` mutation (create mode) or `useUpdateChart` mutation (edit mode). On success: `toast.success('Chart saved')`. In create mode, `useNavigate()` to `/charts`. In edit mode, stay on page.
