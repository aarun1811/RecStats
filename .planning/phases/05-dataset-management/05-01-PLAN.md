## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> FastAPI | Untrusted input from browser to API endpoints |
| FastAPI -> Superset | API calls to Superset for virtual dataset sync |
| FastAPI -> PostgreSQL | Dataset CRUD operations on recviz_datasets table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | managed_datasets.py POST/PUT | mitigate | Pydantic v2 validates all request bodies with strict types, min/max lengths on name (1-256) and description (0-1024). ColumnMetaSchema uses Literal types for data_type, role, aggregation. |
| T-05-02 | Tampering | dataset SQL field | accept | SQL is stored as-is and executed by Superset SQL Lab engine which has its own query validation and sandboxing. RecViz does not execute SQL directly. Auth deferred to SECU-01. |
| T-05-03 | Denial of Service | managed_datasets.py POST | mitigate | Pydantic max_length on name (256) and description (1024) prevents oversized metadata. SQL length is unbounded but Superset enforces query timeouts (120s httpx timeout). |
| T-05-04 | Information Disclosure | managed_datasets.py error responses | mitigate | Use sanitize_detail() from app.core.errors for all exception messages. Never expose raw Superset errors or connection URIs to frontend. |
| T-05-05 | Elevation of Privilege | DELETE endpoint | mitigate | Reference check prevents deleting datasets used by charts (409 response). Delete guard returns empty now but will enforce when Phase 6 adds charts. |
</threat_model>
