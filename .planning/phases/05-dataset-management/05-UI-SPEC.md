# Phase 5 -- UI Design Contract

> Visual and interaction contract for dataset management: dataset list page, dedicated dataset editor, column metadata editor, Explorer "Save as Dataset" dialog, and sync status handling. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline badge padding, status dot margin |
| sm | 8px (`gap-2`, `p-2`) | Toolbar button gaps, form field internal gaps, card content spacing |
| md | 16px (`gap-4`, `p-4`) | Card grid gaps, dataset card internal padding, editor panel gaps |
| lg | 24px (`gap-6`, `p-6`) | Page padding, dataset editor section spacing, list page sections |
| xl | 32px (`gap-8`) | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Monaco editor internal padding: `{ top: 12, bottom: 12 }` -- matches existing `sql-editor.tsx` pattern
- Dataset editor toolbar height: 36px (`h-9`) -- matches existing SQL Editor toolbar height
- Dataset list toolbar input: `h-8` -- matches existing `data-sources-toolbar.tsx` pattern
- Save as Dataset dialog: `p-6` internal -- standard Shadcn DialogContent default

**Source:** CLAUDE.md (page padding `p-6`, section gaps `gap-6`, grid gaps `gap-4`), existing `data-sources-toolbar.tsx` and `sql-editor.tsx` patterns

## Interaction Contract

### Dataset List Page

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Load dataset list | Page mount | Skeleton grid/rows, then datasets appear | <500ms cached, <2s cold |
| Search datasets | Type in search input | Client-side filter on name and description | 0ms (instant filter) |
| Filter by database | Select from database dropdown | Client-side filter by database_id | 0ms (instant filter) |
| Switch view mode | Click grid/list toggle | View switches between card grid and row list | <16ms |
| Click dataset card/row | Click card or row | Navigate to `/datasets/{id}/edit` | Standard route transition |
| Click "+ New Dataset" | Click button | Navigate to `/datasets/new` | Standard route transition |
| Hover dataset card | Mouse enters card | Card lifts with shadow (`hover:shadow-md hover:-translate-y-0.5`) | <16ms (CSS transition) |

### Dataset Editor Page

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Load existing dataset | Page mount with `$datasetId` | Load dataset from API, populate all fields, Monaco editor loads SQL | <1s |
| Edit dataset name | Type in name input | Local state update | 0ms |
| Edit dataset description | Type in description input | Local state update | 0ms |
| Select database | Choose from dropdown | Database selection updates, schema browser context changes | 0ms |
| Edit SQL | Type in Monaco editor | Local state update, "SQL re-run required" banner appears if SQL differs from last-run SQL | 0ms |
| Run query (Cmd+Enter or click) | Click "Run Query" or press Cmd+Enter | Loading state on button, query executes via `/api/sql/execute`, results appear in preview grid, column metadata auto-detected/merged | Standard query latency |
| Column metadata merge | After query re-run with changed columns | New columns added with auto-detected defaults, removed columns flagged red as "missing" | <100ms after results arrive |
| Edit column display name | Click cell in AG Grid metadata table | Cell enters edit mode (AG Grid inline editing) | <16ms |
| Edit column type | Click type cell in metadata table | Dropdown selector appears with string/number/date/currency options | <16ms |
| Edit column role | Click role cell | Dropdown selector appears with dimension/measure/time/none options | <16ms |
| Edit column aggregation | Click aggregation cell | Dropdown selector appears with None/SUM/AVG/COUNT/MIN/MAX/COUNT DISTINCT | <16ms |
| Edit column format | Click format cell | FormatPresetSelect dropdown appears with presets + custom input | <16ms |
| Toggle "Show formatted" | Click toggle button in results preview toolbar | Results grid re-renders with formatted values applied | <200ms |
| Save dataset | Click "Save Dataset" button | Dataset saves to RecViz DB, syncs to Superset, toast confirms, navigate to list (create) or stay (edit) | <2s |
| Save with changed SQL not re-run | Click "Save Dataset" | Button is disabled, banner shows "Run the query before saving" | 0ms (prevented) |
| Delete dataset | Click delete button in editor header | If no charts reference it: confirmation dialog. If charts reference: blocked dialog listing referencing charts. | <500ms to check references |

### Explorer "Save as Dataset" Flow

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Click "Save as Dataset" | Click button in results toolbar | Dialog opens with name, description, database (auto-selected from current Explorer DB) fields | <16ms |
| Enter name and save | Fill name, click "Save" | Dataset created with auto-detected column metadata, dialog closes, toast confirms with link to edit | <2s |
| Cancel dialog | Click Cancel or press Escape | Dialog closes, no side effects | <16ms |

### Delete Dataset Flow

| Interaction | Gesture | Response |
|-------------|---------|----------|
| Click Delete (no references) | Click delete in editor toolbar | Confirmation dialog: "Delete {name}? This cannot be undone." with Cancel and "Delete Dataset" buttons |
| Confirm delete | Click "Delete Dataset" in dialog | Dataset deleted, toast confirms, navigate to `/datasets` list |
| Click Delete (has references) | Click delete in editor toolbar | Blocked dialog: "Cannot delete {name}" with list of referencing charts. Only a "Close" button. |

#### Column Definitions

| Column | Width | Editable | Cell Editor | Notes |
|--------|-------|----------|-------------|-------|
| Name | 120px | No (read-only) | -- | `font-mono text-xs`, original SQL column name |
| Display Name | 140px | Yes | Text input | Default: auto-detected from D-11 |
| Type | 100px | Yes | Select dropdown | Options: `string`, `number`, `date`, `currency` |
| Role | 100px | Yes | Select dropdown | Options: `dimension`, `measure`, `time`, `none` |
| Aggregation | 120px | Yes | Select dropdown | Options: `None`, `SUM`, `AVG`, `COUNT`, `MIN`, `MAX`, `COUNT DISTINCT` |
| Format | flex | Yes | FormatPresetSelect | Preset dropdown + custom input |

#### Auto-Detection Rules (D-11)

| Detection | Rule |
|-----------|------|
| Type: number | `typeof value === 'number'` from first 10 rows |
| Type: date | Column name matches `*_date`, `*_time`, `*_at`, `*_on` |
| Type: string | Default fallback |
| Role: time | Type is date |
| Role: measure | Type is number |
| Role: dimension | Type is string |
| Display name | `column_name.replace(/_/g, ' ')` then capitalize each word |

#### Column Diff Behavior (D-16)

When SQL is re-run and columns change:
- **New columns:** Added to bottom of metadata table with auto-detected defaults, highlighted with `bg-green-50 dark:bg-green-950/20` for one render cycle
- **Removed columns:** Row stays visible with `bg-red-50 dark:bg-red-950/20` background, "Missing" badge in Name column, and a dismiss button (X) to acknowledge removal
- **Unchanged columns:** Retain all existing metadata -- no changes
