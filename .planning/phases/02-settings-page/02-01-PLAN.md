**Create `frontend/src/stores/display-store.ts`** (per D-01, D-02, D-03):

Define the store interface and create the Zustand store:

```typescript
interface DisplayStore {
  density: 'comfortable' | 'compact'
  fontSize: 'small' | 'medium' | 'large'
  setDensity: (d: 'comfortable' | 'compact') => void
  setFontSize: (s: 'small' | 'medium' | 'large') => void
}
```

The store MUST:
1. Use `create<DisplayStore>()` from `zustand` (same pattern as filter-store.ts, builder-store.ts)
2. Read from `localStorage` key `recviz-display` on init to restore persisted values. Default: `density: 'comfortable'`, `fontSize: 'medium'`
3. On init AND on every `setDensity`/`setFontSize` call, write CSS variables to `:root` via `document.documentElement.style.setProperty()`:

   **Density mapping:**
   - `comfortable`: `--spacing-scale: 1`, `--row-height: 36px`, `--cell-padding: 12px`
   - `compact`: `--spacing-scale: 0.85`, `--row-height: 28px`, `--cell-padding: 8px`

   **Font size mapping:**
   - `small`: `--font-scale: 0.875`, `--base-font-size: 12.25px`
   - `medium`: `--font-scale: 1`, `--base-font-size: 14px`
   - `large`: `--font-scale: 1.125`, `--base-font-size: 15.75px`

4. Write to `localStorage` key `recviz-display` on every state change (JSON.stringify `{ density, fontSize }`)
5. Export a helper `initDisplayStore()` function that reads localStorage and writes CSS vars -- call this at store creation time (not lazily)
6. Named export: `useDisplayStore`
7. Follow project conventions: `from __future__` not needed (TS), named function declarations where possible, `import type` for type-only imports

Find the `@layer base` section (or create one if needed). Add to the `body` rule:

body {

This is from UI-SPEC "Transition Rule Addition" section. It enables smooth visual transitions when density/font-size CSS variables change.
