1. Type a forbidden SQL: `DROP TABLE test` in the editor
2. Click Run
3. Verify an error message appears (the API returns 400 for read-only violations)
4. Verify the error mentions "read-only" or "SELECT" or "only SELECT"

**Important notes for SQL Explorer tests:**
- The SQL editor is a Monaco editor. To type into it:
  - Click on the editor container to focus it
  - Use `page.keyboard.type()` after focusing, or find the textarea element within Monaco
  - Alternatively, use `page.evaluate` to set the editor value via Monaco's API if direct typing is unreliable
- The Run button may have an icon (Play/triangle) rather than text -- read the sql-editor component to identify the exact button
- Read-only check: SQL Explorer uses `database_id: ""` by default (empty string). Make sure the test either selects a database from the dropdown first, or the backend handles empty database_id gracefully. Read the sql.py backend code to confirm. If a database_id is required, the test should select a database first.
- NOTE: The default SQL in the explorer is `SELECT * FROM breaks WHERE desk = 'Operations' LIMIT 20` -- this requires a seeded database with the breaks table. For a simpler test, use `SELECT 1 AS test_col` which works on any PostgreSQL connection.
