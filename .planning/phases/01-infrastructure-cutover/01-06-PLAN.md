```bash
docker ps --filter name=oracle-26ai --format "{{.Status}}"
```

If not running, start it:

Wait for the container to be healthy (the gvenzl/oracle-free image reports health status). If container doesn't exist at all, the user must create it first — this is a prerequisite.

```bash
# Check if .env exists
cat backend/.env 2>/dev/null
```

```
RECVIZ_DB_URL=oracle+oracledb://recviz:recviz_dev@localhost:1521/FREEPDB1
ORACLE_CLIENT_LIB_DIR=/Users/aarun/oracle/instantclient
RECVIZ_ENCRYPTION_KEY=ZtmS2OQUhct4iBQmAcreQftJoeodRw4h7Rz3fU8ZPG4=
VITE_API_BASE_URL=http://localhost:8000
```

NOTE: Use the user's actual ORACLE_CLIENT_LIB_DIR path. Check `ls ~/oracle/instantclient/libclntsh.dylib` to verify.

**Step 2: Grant permissions to recviz user (if needed):**

The Docker Oracle container's APP_USER may not have v$session_connect_info access. Connect as ADMIN and grant:

```bash
docker exec -i oracle-26ai sqlplus -S system/RecViz2026@FREEPDB1 <<'SQL'
GRANT SELECT ON v_$session_connect_info TO recviz;
GRANT CREATE TABLE TO recviz;
GRANT UNLIMITED TABLESPACE TO recviz;
SQL
```

Note: Oracle uses `v_$session_connect_info` (with underscore) for granting on v$ views.

**Step 3: Run alembic upgrade head:**

```bash
cd backend && \
  RECVIZ_DB_URL=oracle+oracledb://recviz:recviz_dev@localhost:1521/FREEPDB1 \
  ORACLE_CLIENT_LIB_DIR=$HOME/oracle/instantclient \
  RECVIZ_ENCRYPTION_KEY=ZtmS2OQUhct4iBQmAcreQftJoeodRw4h7Rz3fU8ZPG4= \
  alembic -c app/migrations/alembic.ini upgrade head
```

Verify: Should output "Running upgrade  -> 001, initial oracle schema" (or similar). Check tables:

```bash
docker exec -i oracle-26ai sqlplus -S recviz/recviz_dev@FREEPDB1 <<'SQL'
SELECT table_name FROM user_tables ORDER BY table_name;
SQL
```

Expected output should list: `RECVIZ_ALEMBIC_VERSION`, `RECVIZ_CHARTS`, `RECVIZ_CONNECTIONS`, `RECVIZ_DASHBOARDS`, `RECVIZ_DATASETS`, `RECVIZ_DATA_SOURCES`, `RECVIZ_KPIS` (7 tables including alembic version).

**Step 4: Boot backend:**

```bash
cd backend && \
  uvicorn app.main:app --reload --port 8000 &
```

Wait a few seconds for startup, then:

```bash
curl -s http://localhost:8000/health | python3 -m json.tool
```

Expected: `{"status": "healthy", "driver": "python-oracledb", "mode": "thick"}`

Check the startup log for:
- `oracledb X.Y.Z thick mode initialized`
- `Oracle client driver: python-oracledb thk` (NOT thn)
- No warnings about thin mode fallback

**Step 5: Verify frontend compiles:**

```bash
cd frontend && npx tsc --noEmit
```

Should exit 0 with no errors. The palette changes in index.css should not affect TypeScript compilation.

**Step 6: Verify frontend loads:**

```bash
cd frontend && pnpm dev &
```

Wait for Vite to report `Local: http://localhost:5173/`, then open in browser. The app should load without a blank white screen. Sidebar should show the mist-tinted background (not pure gray). Primary buttons should be blue (not black).

If any step fails, diagnose and fix. Common issues:
- Oracle container not running → `docker start oracle-26ai`
- Missing grants → run the GRANT statements above
- ORACLE_CLIENT_LIB_DIR wrong path → check `ls $HOME/oracle/instantclient/libclntsh.dylib`
- Migration fails → check for typos in migration DDL, verify ORM models import correctly
