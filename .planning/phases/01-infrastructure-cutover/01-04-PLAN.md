**chart-themes.ts** — Rewire per D-17, INFRA-21. Addresses Gemini LOW concern about getComputedStyle timing.

**First: Add hex fallbacks to resolveColor().** The existing `resolveColor()` function reads CSS vars via `getComputedStyle()`. If called before the DOM is fully painted (e.g., during SSR or early effect execution), `getComputedStyle` may return empty strings, causing charts to disappear. Add a fallback map so charts always have valid colors.

Add a `HEX_FALLBACKS` constant near the top of the file (after existing utility functions):

```typescript
/**
 * Hard-coded hex fallbacks for CSS variables, used when getComputedStyle
 * returns empty (e.g., DOM not yet painted, headless test environment).
 * Addresses review concern: LOW — getComputedStyle timing safety.
 */
const HEX_FALLBACKS: Record<string, string> = {
  '--series-1': '#93b8e8',
  '--series-2': '#4d7adb',
  '--series-3': '#3a5fc9',
  '--series-4': '#3050b8',
  '--series-5': '#2641a0',
  '--series-6': '#2fb898',
  '--series-7': '#d4a030',
  '--series-8': '#8b5cf6',
  '--color-ramp-low': '#93b8e8',
  '--color-ramp-high': '#2641a0',
  '--chart-positive': '#22c55e',
  '--chart-negative': '#ef4444',
  '--primary-foreground': '#f0f0ff',
}
```

Then modify the existing `resolveColor` function to use the fallback when `getComputedStyle` returns empty or fails:

```typescript
function resolveColor(cssVar: string): string {
  const raw = getCssVar(cssVar)
  if (!raw || raw.trim() === '') {
    // Fallback: DOM not yet painted or variable undefined
    return HEX_FALLBACKS[cssVar] ?? '#888888'
  }
  return cssColorToHex(raw)
}
```

**Note:** If the current `resolveColor` signature or implementation differs, adapt the fallback logic to fit. The key requirement is: if the CSS variable read returns empty/undefined, return the hex fallback instead of an empty string.
