    - Parameters: `db_id: str, session: DbSessionDep, page: int = 1, page_size: int = 50`
    - Query recviz_datasets WHERE database_id matches (need to handle that database_id in recviz_datasets is int -- but connections now use string UUIDs, so this endpoint should match by connection name or we need to adapt). IMPORTANT: The recviz_datasets.database_id is an int (Superset-era). Since we haven't migrated that field yet (it maps to Superset DB IDs, not connection UUIDs), this endpoint should return an empty list or match by name lookup. For now, just return the paginated list of datasets from recviz_datasets where database_id matches. Since the old Superset int IDs won't match UUID strings, this endpoint will naturally return empty until datasets are updated. This is acceptable -- the endpoint shape is preserved.

    **POST "" (create_database):**
    - Parameters: `body: DatabaseCreate, session: DbSessionDep, encryption: EncryptionDep, engine_manager: EngineManagerDep, request: Request`
    - Generate UUID with `str(uuid.uuid4())`
    - Encrypt password: `encryption.encrypt(body.password or "")`
    - Create RecvizConnection record with all fields from body
    - `session.add(connection)` then `await session.flush()` (session auto-commits via dependency)
    - Invalidate ConnectionResolver: `resolver = request.app.state.connection_resolver; await resolver.invalidate(session)`
    - Return 201 with response shape matching DatabaseInfo

    **PUT "/{db_id}" (update_database):**
    - Parameters: `db_id: str, body: DatabaseUpdate, session: DbSessionDep, encryption: EncryptionDep, engine_manager: EngineManagerDep, request: Request`
    - Look up connection by id, 404 if not found
    - Apply non-None fields from body to the connection record
    - If password changed: `connection.encrypted_password = encryption.encrypt(body.password)`
    - Dispose old engine: `await engine_manager.dispose_engine(db_id)`
    - Invalidate ConnectionResolver cache: `await resolver.invalidate(session)`
    - Return updated record in same shape

    **DELETE "/{db_id}" (delete_database):**
    - Parameters: `db_id: str, session: DbSessionDep, engine_manager: EngineManagerDep, request: Request`
    - Look up connection by id, 404 if not found
    - `await session.delete(connection)`
    - `await engine_manager.dispose_engine(db_id)`
    - Invalidate ConnectionResolver
    - Return `{"success": True}`

    **POST "/test" (test_connection):**
    - Parameters: `body: TestConnectionRequest, request: Request`
    - Build async URI from body fields using `build_async_uri()`
    - Call `EngineManager.test_connection(uri, body.backend)` (static method, no instance needed)
    - If `body.database_id` provided, update status tracker: `mark_connected` or `mark_unreachable`
    - Return `{"success": bool, "message": str}`

    **Remove the POST "/{db_id}/sync" endpoint entirely** -- Superset dataset sync is being removed. If keeping for API compatibility, make it return `{"success": True, "dataset_count": 0}` as a no-op.

    Error handling: Replace all httpx error handling with SQLAlchemy exception handling. Catch `sqlalchemy.exc.IntegrityError` for duplicate name on create. Catch `ValueError` from `build_async_uri` for invalid backend.
