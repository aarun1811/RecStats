<objective>
FilterConfigDialog for adding dashboard filters from dataset columns, and BuilderFilterBar for managing filter order in edit mode.

Purpose: Filters are how users narrow dashboard data (D-20). The FilterConfigDialog lets users pick columns from datasets already used on the dashboard, auto-detects filter types (D-20), supports cascading (D-22), and allows manual column mapping override (D-21). The BuilderFilterBar renders filters above the canvas (D-24) with drag-to-reorder (D-23) and remove.

Output: FilterConfigDialog, BuilderFilterBar, wired into BuilderPage.
</objective>

Behavior per D-20, D-21, D-22, D-28:

1. **Gather datasets used by charts on the dashboard**: Read `items` from builder store. For chart items, extract `item.chart.datasetId`. For grid items, extract `item.grid.datasetId`. Deduplicate. Fetch each dataset via `useManagedDatasets()` (bulk fetch, filter client-side).

2. **Display datasets as expandable sections**: Each dataset shows its name. Below, list its columns as checkbox rows.

3. **Each column row**:
   - `<Checkbox>` for selection
   - Column name: `displayName` (or `name` fallback)
   - Auto-detected filter type badge:
     - `dataType === 'string'` or `role === 'dimension'` -> `<Badge variant="secondary">Multi Select</Badge>`
     - `dataType === 'date'` or `role === 'time'` -> `<Badge variant="secondary">Date Range</Badge>` (note: preset-range maps to this conceptually; we use 'preset-range' FilterConfig type)
     - `dataType === 'number'` or `dataType === 'currency'` and `role === 'measure'` -> `<Badge variant="secondary">Number Range</Badge>` (use 'preset-range' type)
   - Override dropdown: `<Select>` to change filter type (Single Select, Multi Select, Date Range, Number Range)
   - Each selected column gets an optional "Depends on" dropdown (D-22): lists other already-selected filter columns. Maps to `FilterOptionsSource.dependsOn`.

4. **On "Add Filters" click**: Build `FilterConfig[]` from selected columns:
   - `id`: `crypto.randomUUID()`
   - `label`: column `displayName` or `name`
   - `type`: auto-detected or user-overridden ('single-select' | 'multi-select' | 'preset-range')
   - `lockable`: false
   - `optionsSource`: `{ dataSourceId: dataset.id (per Research Item 1 — first dataset containing the column), valueColumn: column.name, dependsOn: {} or { parentFilterId: parentColumnName } }`
   Call `onFiltersAdded(filters)`.

5. **UI layout per UI-SPEC**: Dialog max-w-lg, ScrollArea, footer with [Add Filters] button.

6. **Skip columns already added as filters**: Check existing `filters` in builder store, grey out or hide columns that already have a matching filter.
