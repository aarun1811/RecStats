## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, grid item internal micro-spacing |
| sm | 8px | Compact element spacing, toolbar button gaps |
| md | 16px | Default element spacing, react-grid-layout gap (`gap-4`), card internal padding |
| lg | 24px | Section padding, page padding (`p-6`) |
| xl | 32px | Major section gaps (filter bar to canvas) |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- react-grid-layout rowHeight: 80px (D-06). Not a spacing token -- it is a grid configuration value.
- react-grid-layout margin: `[16, 16]` (matches `gap-4` / md token).
- react-grid-layout containerPadding: `[0, 0]` (canvas fills available space; page padding handled by parent).

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Page title | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold tracking-tight` |
| Section title / Panel header | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Body / Labels | 14px | 400 (regular) | 1.5 | `text-sm` |
| Caption / Metadata | 12px | 400 (regular) | 1.5 | `text-xs text-muted-foreground` |

**3-weight exception:** This phase uses 3 font weights (400, 500, 600) which exceeds the standard 2-weight limit. This is an intentional project-level override aligned with CLAUDE.md conventions established across all prior phases: `font-semibold` (600) for page titles and primary headings, `font-medium` (500) for section headers, panel titles, and toolbar button labels, and `font-normal` (400) for body text, descriptions, and captions. Reducing to 2 weights would break consistency with the existing codebase typography hierarchy.

Notes:
- Dashboard title on canvas: `text-2xl font-semibold tracking-tight` (inline-editable in edit mode, static in view mode).
- Dashboard description: `text-sm text-muted-foreground` (inline-editable in edit mode).
- Edit toolbar button labels: `text-sm font-medium`.
- Panel title bars: `text-sm font-medium` (matches existing `CardTitle` in config-chart-grid).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / oklch(1 0 0) | Page background, canvas area |
| Secondary (30%) | `var(--card)` / oklch(1 0 0) light, `var(--muted)` | Chart panel cards, KPI cards, filter bar, toolbar background |
| Accent (10%) | `var(--primary)` / oklch(0.205 0 0) | See reserved-for list below |
| Destructive | `var(--destructive)` / oklch(0.577 0.245 27.325) | Delete Dashboard button, Remove panel button |

Accent reserved for:
- **Save Dashboard button** (primary variant) in edit toolbar
- **[+ Add] button** (primary variant) in edit toolbar
- **Selected panel outline** (blue ring: `ring-2 ring-primary/50`) during edit mode hover
- **Active drag placeholder** outline (dashed `border-primary/40`)
- **Create Dashboard** button on list page

Additional semantic colors:
- Edit mode canvas dashed grid lines: `border-dashed border-border/40` (uses `--border` CSS variable at 40% opacity)
- Panel drag handle hover: `text-muted-foreground` transitioning to `text-foreground`
- Unsaved changes indicator: `bg-amber-500` dot (2.5px, `size-2.5 rounded-full`)

All colors reference Shadcn CSS variables exclusively. No hardcoded hex/rgb/hsl. Dark mode variants handled automatically through the oklch CSS variable system.

## Layout Specifications

### Dashboard List Page (`/dashboards`)

**Focal Point:** The [+ Create Dashboard] primary button anchored top-right of the toolbar. On empty state, the focal point shifts to the centered [+ Create Dashboard] button inside the Empty component.

```
+------------------------------------------------------------------+
| Page Title: "Dashboards"                    [Search] [Filters] [Toggle] [+ Create Dashboard] |
+------------------------------------------------------------------+
| Grid: 3-col (xl:3, lg:2, md:1)  OR  List: single-column rows    |
| +------------------+ +------------------+ +------------------+   |
| | Dashboard Card   | | Dashboard Card   | | Dashboard Card   |   |
| | Name             | | Name             | | Name             |   |
| | Description      | | Description      | | Description      |   |
| | Last modified    | | Last modified    | | Last modified    |   |
| +------------------+ +------------------+ +------------------+   |
+------------------------------------------------------------------+
```

- Page padding: `p-6`
- Card grid gap: `gap-4`
- Cards: Shadcn `Card` with `CardHeader` containing title, description, metadata
- Card hover: `hover:shadow-lg hover:shadow-primary/5 hover:-translate-y-0.5 transition-all duration-200`
- Card click navigates to `/dashboards/:id` (view mode)
- Toolbar matches chart library toolbar pattern: search input (max-w-xs, h-8), view toggle (List/Grid), create button (primary, h-8)
- Empty state: `Empty` component with `LayoutDashboard` icon, "No dashboards yet" title, "Create your first dashboard..." body, [+ Create Dashboard] button

### Edit Mode (`/dashboards/:id/edit` or `/dashboards/new`)

**Focal Point:** The canvas area with its dashed grid background. When the canvas is empty, the focal point is the centered Empty component with the "+ Add Content" CTA. When panels are present, the focal point is the panel content area. The [+ Add] button in the toolbar is the secondary focal point for continuing to build.

```
+------------------------------------------------------------------+
| BuilderToolbar: [+ Add v] | [Undo] [Redo] | [Save Dashboard] [Save As] [Exit Builder] |
+------------------------------------------------------------------+
| Dashboard Title (inline editable)                                 |
| Dashboard Description (inline editable)                           |
+------------------------------------------------------------------+
| Filter Bar (if filters configured)            [+ Add Filter] drag handles |
+------------------------------------------------------------------+
| Canvas (react-grid-layout, 12-col, dashed grid bg)               |
|                                                                    |
|  +--BuilderPanel--+  +--BuilderPanel--+  +--BuilderPanel--+      |
|  | [drag] Title [edit][x] | [drag] Title [edit][x] |              |
|  |                 |  |                 |                          |
|  | Live Chart      |  | Live KPI       |  | Live Grid             |
|  |                 |  |                 |  |                       |
|  +---resize------+  +---resize------+  +---resize------+         |
|                                                                    |
+------------------------------------------------------------------+
```

- Full-page width, no side panel (D-10)
- Toolbar: sticky at top, `bg-background border-b`, `px-6 py-2`, `flex items-center justify-between`
- Toolbar left: [+ Add] dropdown (primary button, `size="sm"`), separator, Undo/Redo icon buttons (ghost variant, `size="sm"`)
- Toolbar right: [Save Dashboard] (primary, `size="sm"`), [Save As] (outline, `size="sm"`), [Exit Builder] (ghost, `size="sm"`)
- Canvas background: repeating dashed grid pattern via CSS background-image (12-column visual guide, `border-dashed border-border/40`)
- Canvas padding: `p-0` (react-grid-layout handles internal spacing via margin config)
- Inline title: `text-2xl font-semibold tracking-tight`, input with `border-transparent hover:border-input focus:border-input` styling
- Inline description: `text-sm text-muted-foreground`, input with same transparent border pattern

## Interaction Contracts

### Drag and Drop (react-grid-layout)

- 12-column grid, rowHeight 80px, margin [16, 16], containerPadding [0, 0]
- Vertical compaction ON (`compactType: "vertical"`)
- No overlap, push others on collision (`preventCollision: false`, default behavior)
- Drag initiated from GripVertical handle on panel header (`draggableHandle: ".drag-handle"`)
- Resize from bottom-right corner handle
- Minimum sizes enforced: Charts 3w x 3h, KPI 2w x 2h, Grid 6w x 4h
- Drag animation: react-grid-layout default CSS transition (200ms)
- Drop placeholder: dashed border `border-2 border-dashed border-primary/40 rounded-lg bg-primary/5`

### Undo/Redo

- Keyboard: Ctrl+Z (undo), Ctrl+Shift+Z (redo)
- Toolbar buttons: `Undo2` and `Redo2` icons (Lucide), ghost variant, disabled when stack empty
- Snapshots: layout JSON array captured after each layout change, add, remove, or config change
- Stack depth: 50 entries max, oldest entries discarded
- Stack reset: cleared on Save

### Mode Transitions

- View to Edit: click [Edit] button in view mode header, navigates to `/dashboards/:id/edit`
- Edit to View: [Exit Builder] returns to `/dashboards/:id` (with unsaved changes warning if dirty), or [Save Dashboard] saves then navigates to `/dashboards/:id`
- New dashboard: `/dashboards/new` enters edit mode immediately with empty canvas
- Page transition animation: `motion.div` with `opacity: 0 -> 1`, `duration: 0.2`, `ease: easeOut` (matches existing chart/KPI page pattern)

### Unsaved Changes

- Track dirty state: compare current layout/config JSON against last saved version
- On navigate away (Exit Builder, browser back, sidebar link): show `Dialog` with unsaved changes warning
- Dialog copy: "You have unsaved changes. Leave without saving?"
- Dialog actions: [Keep Editing] (outline) and [Leave Without Saving] (destructive variant)
- On browser tab close: `beforeunload` event with standard browser warning
- No auto-save (too complex for v1, user expectation is explicit Save)

### Filter Bar in Edit Mode

- Filter bar renders at top of canvas, outside react-grid-layout (D-24)
- Each filter chip has a small `GripVertical` handle for drag-to-reorder
- Drag-to-reorder uses HTML5 drag or a simple sortable list (not react-grid-layout)
- [x] remove button on each filter chip in edit mode
- [+ Add Filter] button at end of filter bar opens FilterConfigDialog

### Panel Content Rendering

- Charts render live data inside builder panels (D-08 WYSIWYG)
- Chart content is interactive during edit mode (tooltips work, but cross-filter clicks do NOT trigger in edit mode)
- KPI cards render with real computed values (using KpiPreviewCard)
- Data grids render with AG Grid, limited to panel viewport height
- Loading state: Skeleton placeholder inside panel while data loads
- Error state: ErrorPanel component inside panel

## Responsive Behavior

Desktop-first application. No mobile/tablet optimization.

- Minimum supported width: 1280px (breakpoint: `xl`)
- Below 1280px: content still renders but may require horizontal scroll
- react-grid-layout width: 100% of container (auto-calculated from parent)
- Dashboard list: 3 columns at xl, 2 at lg, 1 at md (matches existing chart/KPI library pattern)

## Dark Mode

Every component in this phase works in both light and dark mode:

- Canvas dashed grid: `border-border/40` auto-adjusts via CSS variable
- Panel cards: `bg-card` auto-adjusts
- Toolbar: `bg-background border-b` auto-adjusts
- Dialog backdrops: Shadcn default dark overlay

- Unsaved changes dot: `bg-amber-500` works in both modes (no dark variant needed)
- All text colors use semantic Tailwind classes (`text-foreground`, `text-muted-foreground`, etc.)
