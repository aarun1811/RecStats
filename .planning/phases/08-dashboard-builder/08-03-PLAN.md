If the v2 API differs from research (e.g., props are flat instead of config objects, or there is no `useContainerWidth` hook), adapt accordingly — use the actual API, not the research assumptions.

3. Create `frontend/src/components/builder/builder-canvas.tsx`:

   Key configuration per decisions:
   - 12 columns (D-01)
   - rowHeight: 80 (D-06)
   - margin: [16, 16] (D-06, matches gap-4)
   - containerPadding: [0, 0] (UI-SPEC)
   - Vertical compaction ON (D-04) — use the correct v2 API for this (may be `compactType="vertical"`, `compactor={verticalCompactor}`, or similar — verify from types)
   - No overlap, push others (D-05) — default collision behavior
   - Drag handle: `.drag-handle` CSS selector (D-03)
   - Resize enabled (D-03)

   Minimum sizes per item type (D-02):
   - chart: minW=3, minH=3
   - kpi: minW=2, minH=2
   - grid: minW=6, minH=4

   Layout mapping functions:
   - `toRglLayout(items: BuilderItem[]): Layout` — map each item to `{ i: item.id, x: item.layout.col, y: item.layout.row, w: item.layout.width, h: item.layout.height, minW: getMinWidth(item.type), minH: getMinHeight(item.type) }`
   - `fromRglLayout(rglLayout: LayoutItem[]): Array<{id: string; layout: ChartLayout}>` — map back

   onLayoutChange handler:
   - Compare incoming layout with current store items to detect actual changes (prevent ping-pong loop per Pitfall 3)
   - If changed: push current layouts to history store, then update builder store

   Canvas background (UI-SPEC):
   - Dashed grid pattern via CSS: `className="relative min-h-[400px]"` with `style={{ backgroundImage: 'radial-gradient(circle, hsl(var(--muted-foreground) / 0.15) 1px, transparent 1px)', backgroundSize: '20px 20px' }}`

   Import both required CSS files at top of file:
   ```typescript
   import 'react-grid-layout/css/styles.css'
   import 'react-resizable/css/styles.css'
   ```

   If v2 provides `useContainerWidth()` hook, use it and gate rendering on `mounted`. If not available, use a ResizeObserver or ref-based width measurement.

   Props interface:
   ```typescript
   interface BuilderCanvasProps {
     children: React.ReactNode
   }
   ```

   The component reads items from `useBuilderStore` directly (no prop drilling).
