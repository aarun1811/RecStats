This component renders live data inside builder panels per D-08 ("Real data renders inside charts while editing"). It uses the existing renderer components — ChartFactory, KpiPreviewCard — which already work. The builder panels must show real data, not placeholder cards.

```typescript
interface BuilderPanelContentProps {
  item: BuilderItem
}

export function BuilderPanelContent({ item }: BuilderPanelContentProps) {
  switch (item.type) {
    case 'chart':
      return <BuilderChartContent chartRef={item.chart!} itemId={item.id} />
    case 'kpi':
      return <BuilderKpiContent kpiRef={item.kpi!} />
    case 'grid':
      return <BuilderGridContent gridRef={item.grid!} />
    default:
      return null
  }
}
```

**BuilderChartContent** (private helper in same file):
- Takes `chartRef: BuilderChartRef` and `itemId: string`
- Uses `useDataSourceQuery(chartRef.datasetId, {}, true)` to fetch data (empty filters in builder context — no filter bar active during editing)
- Converts query response to `ChartDataResponse` (follow the exact pattern from `config-chart-grid.tsx` `QueryChartItemWithDrill` — normalize columns)
- Builds a `ChartConfig` from the `BuilderChartRef`:
  ```typescript
  const chartConfig: ChartConfig = {
    id: itemId,
    name: chartRef.title,
    vizType: chartRef.chartType,
    datasourceId: 0,
    metricColumns: [], // resolved at render time by ChartFactory
  }
  ```
- Renders `<ChartFactory chartId={itemId} config={chartConfig} data={chartData} isLoading={isLoading} error={error ?? null} className="h-full w-full" />`
- Shows `<Skeleton className="h-full w-full" />` while loading

**BuilderKpiContent** (private helper in same file):
- Takes `kpiRef: BuilderKpiRef`
- Uses `useManagedKpi(kpiRef.kpiId)` to fetch the full KPI definition (RecvizKpi) which includes `config: KpiLibraryConfig` with format, trend, thresholds, subtitle
- Uses `useDataSourceQuery(kpi.datasetId, {}, !!kpi)` to fetch data for the KPI metric
- Computes value from data: find the metric column value using `kpi.metricColumn` and `kpi.aggregation` — if the query returns rows, extract the metric. For simple cases, take the first row's value for `kpi.metricColumn`. For aggregation, the backend query already aggregates.
- Renders `<KpiPreviewCard name={kpiRef.title} value={computedValue} isLoading={isLoading} format={kpi.config.format} trend={kpi.config.trend} trendPercentage={null} thresholds={kpi.config.thresholds} subtitle={kpi.config.subtitle} className="h-full border-0 p-3" />`
- Shows skeleton while loading: `<Skeleton className="h-full w-full" />`
- If kpi data not yet loaded, show KpiPreviewCard with `isLoading={true}` and value=0

**BuilderGridContent** (private helper in same file):
- Takes `gridRef: BuilderGridRef`
- Uses `useDataSourceQuery(gridRef.datasetId, {}, true)` to fetch data
- Renders a simplified AG Grid preview (not the full ConfigDataGrid which needs dashboard context):
  ```tsx
  // Show a simple table preview with the dataset data
  <div className="h-full w-full overflow-auto">
    {isLoading ? (
      <Skeleton className="h-full w-full" />
    ) : data ? (
      <div className="text-xs">
        <div className="sticky top-0 bg-muted/80 backdrop-blur flex gap-px">
          {columns.slice(0, 6).map(col => (
            <div key={col} className="px-2 py-1.5 font-medium text-muted-foreground truncate min-w-[100px]">
              {col}
            </div>
          ))}
        </div>
        {data.rows.slice(0, gridRef.rowLimit).slice(0, 10).map((row, i) => (
          <div key={i} className="flex gap-px border-t">
            {columns.slice(0, 6).map(col => (
              <div key={col} className="px-2 py-1 truncate min-w-[100px]">
                {String(row[col] ?? '')}
              </div>
            ))}
          </div>
        ))}
        {data.rowCount > 10 && (
          <div className="px-2 py-1 text-muted-foreground">{data.rowCount} total rows</div>
        )}
      </div>
    ) : null}
  </div>
  ```
