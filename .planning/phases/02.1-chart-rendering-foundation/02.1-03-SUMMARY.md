## Decisions Made
- Created `tsconfig.e2e.json` as a separate TypeScript project reference rather than adding E2E files to `tsconfig.node.json`. This keeps E2E test files isolated with their own compilation scope and avoids mixing test types with build tooling configs.
- Playwright webServer uses `reuseExistingServer: true` so developers with `pnpm dev` already running won't get port conflicts. The backend, Superset, and PostgreSQL must be started separately.
- Canvas click coordinates in cross-filter and drill-down tests are approximate (left-center at 30% width, 50% height). These tests verify the interaction infrastructure works when a data point is hit, but cannot guarantee hitting a data point every time. Manual visual validation (Task 2) is the definitive check.
