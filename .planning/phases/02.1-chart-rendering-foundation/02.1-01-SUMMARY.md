## Decisions Made
- Config-driven column resolution follows explicit fallback chain: config.categoryColumn > first non-metric column > columns[0]. This ensures config metadata always takes priority while gracefully handling configs that omit explicit category.
- buildSeries() returns null for unsupported chart types instead of silently rendering as bar chart. ChartFactory checks SUPPORTED_AG_TYPES before routing, so unsupported types never reach AG Charts.
- Sankey, graph, and parallel ECharts types retain position-based column mapping because their data sources are specifically shaped for the chart type (source/target/value columns).
- Scatter chart uses metricColumns for xKey/yKey/sizeKey (no category concept) with fallback to columns[0]/columns[1].
