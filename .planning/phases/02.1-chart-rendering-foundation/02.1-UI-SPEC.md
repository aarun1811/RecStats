## Color

This phase does not introduce new color tokens. It uses the existing Shadcn CSS variable system exclusively.

| Role | CSS Variable | Usage in This Phase |
|------|-------------|---------------------|
| Dominant (60%) | `--background` | Chart container background (transparent for charts), page background |
| Secondary (30%) | `--card` / `--muted` | Chart card surfaces, cross-filter badge bar (`bg-muted/50`) |
| Accent (10%) | `--primary` | NOT used directly in this phase's new components |
| Destructive | `--destructive` | Error panel icon color, error border tint (`border-destructive/20`, `bg-destructive/5`) |
| Warning | `text-amber-500 dark:text-amber-400` | Column-missing warning icon (AlertTriangle) |
| Chart series | `--chart-1` through `--chart-5` + hardcoded series fallbacks | Multi-series chart differentiation |

### Chart-Specific Color Contracts

| Chart Type | Color Behavior |
|------------|----------------|
| Bar/Line/Area/Combo | Series colors from `getChartPalette().series` array (10 colors) |
| Pie/Donut | Same series palette, one color per segment |
| Scatter | First two series colors for x/y visual distinction |
| Heatmap | `colorRange` gradient: 3 stops derived from chart palette (cool to warm) |
| Treemap | `colorRange` gradient: 2 stops (positive green `#43A047` to negative red-orange `#FF5722`) |
| Waterfall | Positive: series[0], Negative: destructive, Total: muted-foreground |
| Cross-filter dimming | Selected: `fillOpacity: 1, strokeWidth: 2`, Dimmed: `fillOpacity: 0.25, strokeWidth: 0` |

Accent reserved for: Primary CTA buttons (none in this phase), links, focus rings. NOT used for chart elements or error states.

Source: `frontend/src/lib/chart-themes.ts`, `frontend/src/index.css` CSS variables, CLAUDE.md color rules, existing `makeItemStyler()` dimming values

## New UI Components in This Phase

This phase introduces two new inline components and modifies existing chart wrappers. No new pages, no new layout structures.

### 1. Column Missing Error Panel

**When shown:** A config-specified metric or category column is not found in the query data response (D-09, D-10).

**Visual specification:**
- Container: `flex flex-col items-center justify-center gap-2 text-center` inside the chart card content area
- Height: matches the chart card's content area (Shadcn Card default, approximately 300px from existing `h-[300px]` pattern)
- Icon: `AlertTriangle` from Lucide, `size-6`, `text-amber-500 dark:text-amber-400`
- Title: `text-sm font-medium text-foreground` -- "Column mapping error"
- Message: `text-xs text-muted-foreground max-w-[300px] text-center` -- "Column(s) '{columnNames}' not found in data source response."
- Available columns: `text-xs text-muted-foreground/70` -- "Available: {comma-separated list}"
- No border, no background tint (lives inside the existing chart Card which provides the border)
- No retry button (this is a config error, not a transient failure)

**Dark mode:** Amber icon shifts to `text-amber-400`. All text colors use CSS variables (auto-adapted).

### 2. Unsupported Chart Type Error Panel

**When shown:** The chart config specifies a vizType that is not in the supported set (D-05).

**Visual specification:**
- Container: Same layout as Column Missing Error Panel (centered in chart card content)
- Icon: `AlertTriangle` from Lucide, `size-6`, `text-muted-foreground`
- Title: `text-sm font-medium text-foreground` -- "Unsupported chart type"
- Message: `text-xs text-muted-foreground max-w-[300px] text-center` -- "Chart type '{vizType}' is not supported. Supported types: bar, line, area, pie, donut, scatter, heatmap, treemap, waterfall, combo."
- No background tint, no destructive coloring (this is informational, not an error in data)
- No retry button

**Dark mode:** All text colors use CSS variables (auto-adapted).

### 3. Chart Card Layout (Existing -- No Changes)

The chart card structure remains as-is from Phase 2:

```
Card
  CardHeader
    CardTitle (chart.title)
    DrillBreadcrumb (when drilling)
  CardContent
    ChartFactory (AG Charts or ECharts)
    OR ColumnMissingError
    OR UnsupportedChartTypeError
    OR ErrorPanel (query failure)
    OR Skeleton (loading)
```

No layout changes. The new error components replace the chart render area within the existing card.

## Interaction Contracts

### Cross-Filter Visual States (Validation Only -- Already Built)

| State | Visual Treatment | Source |
|-------|-----------------|--------|
| No selection | All chart elements at `fillOpacity: 1` | Default |
| Selection active (same chart) | Selected element: `fillOpacity: 1, strokeWidth: 2, offset: 8` (exploded). All others: `fillOpacity: 0.25, strokeWidth: 0` | `makeItemStyler()` |
| Selection active (other charts) | Cross-filtered data shown at full opacity; non-matching rows excluded from data before render | `useCrossFilter` hook |
| Badge bar | `bg-muted/50 rounded-lg px-3 py-1.5`, badges `variant="secondary"`, clear button `variant="ghost" size="sm"` | `CrossFilterBar` |
| Badge bar animation | Entry: `opacity: 0, height: 0` to `opacity: 1, height: auto`, 200ms ease-out. Badge items: scale 0.9 to 1, 150ms | `motion/react` |

### Drill-Down Visual States (Validation Only -- Already Built)

| State | Visual Treatment | Source |
|-------|-----------------|--------|
| Overview (no drill) | No breadcrumb shown | `DrillBreadcrumb` returns null |
| Drilled into level | Breadcrumb: "Overview > {level1} > {level2}". Root and intermediate levels are clickable links. Current level is `BreadcrumbPage` (not clickable) | `DrillBreadcrumb` |
| Detail grid | Full-width `DrillDetailGrid` slides in below chart row with `AnimatePresence` transition | `config-chart-grid.tsx` |
| Detail grid animation | Slide from `opacity: 0, height: 0` to `opacity: 1, height: auto`, 200ms ease-out | `motion/react` |

### Chart Click/Double-Click Discrimination

| Action | Trigger | Behavior |
|--------|---------|----------|
| Single click | Click on chart element, 250ms debounce | Cross-filter: adds/toggles filter on clicked value |
| Double click | Double-click on chart element | Drill-down: navigates to next drill level for clicked value |

Source: Existing 250ms debounce in `ag-chart-wrapper.tsx`

### Heatmap Interaction

| Action | Behavior |
|--------|----------|
| Hover | AG Charts built-in tooltip showing xKey, yKey, colorKey values |
| Click | Cross-filter on the cell's category value (xKey value) |
| Double-click | Drill-down on the cell's category value |

### Treemap Interaction

| Action | Behavior |
|--------|----------|
| Hover | AG Charts built-in tooltip showing label and size value |
| Click | Cross-filter on the tile's label value |
| Double-click | Drill-down on the tile's label value |

## Chart Theme Additions

### Heatmap Theme (New)

Added to `getAgChartsTheme().overrides`:

```
heatmap: {
  series: {
    colorRange: [palette.series[1], palette.series[3], palette.series[4]],
    label: { enabled: false },
    stroke: palette.background,
    strokeWidth: 2,
  },
}
```

- Color range: 3 stops from the existing series palette (cool blue-ish, warm amber, warm orange)
- Cell stroke: background color at 2px to create visual separation between cells
- Labels disabled by default (heatmap cells are often too small for labels)

### Treemap Theme (New)

Added to `getAgChartsTheme().overrides`:

```
treemap: {
  series: {
    colorRange: ['#43A047', '#FF5722'],
    tile: {
      label: { fontSize: 12, minimumFontSize: 9, color: palette.foreground },
      padding: 6,
      gap: 2,
    },
    group: {
      label: { fontSize: 14, color: palette.foreground },
      padding: 8,
      gap: 4,
    },
  },
}
```

- Color range: green to red-orange (financial data convention: good to bad)
- Tile labels: 12px body size, minimum 9px to avoid unreadable text
- Group labels: 14px (one step up from tile)

### Waterfall Theme (New)

Added to `getAgChartsTheme().overrides`:

```
waterfall: {
  series: {
    item: {
      positive: { name: 'Increase' },
      negative: { name: 'Decrease' },
    },
    line: { strokeWidth: 2 },
  },
}
```

- Positive/negative coloring handled by AG Charts native waterfall (uses series palette automatically)
- Connector lines: 2px width for visibility

## Dark Mode Contract

All new components in this phase use Shadcn CSS variables exclusively, which auto-adapt to dark mode:

| Element | Light Mode | Dark Mode |
|---------|-----------|-----------|
| Chart background | `transparent` | `transparent` |
| Chart card | `--card` (white) | `--card` (dark surface) |
| Error panel text | `--muted-foreground` | `--muted-foreground` (auto-lighter) |
| Warning icon | `text-amber-500` | `text-amber-400` |
| Cross-filter bar bg | `--muted` at 50% opacity | `--muted` at 50% opacity (auto-adapted) |
| Chart axis labels | `--muted-foreground` | `--muted-foreground` |
| Chart grid lines | `--border` dashed | `--border` dashed |
| Series colors | `getChartPalette().series` | Same function, reads live CSS vars |

AG Charts and ECharts themes are rebuilt on theme toggle via `getAgChartsTheme()` and `getEChartsTheme()`, which read CSS variables at call time.

## Accessibility Notes

- Error panels use semantic HTML (`<p>` for messages, not `<div>` with text)
- Cross-filter badge bar has `aria-live="polite"` (already implemented)
- Cross-filter remove buttons have `aria-label` with filter context (already implemented)
- Chart elements are not individually keyboard-accessible (AG Charts handles its own accessibility)
- Drill breadcrumb uses Shadcn Breadcrumb which includes proper `nav` + `aria-label` semantics
