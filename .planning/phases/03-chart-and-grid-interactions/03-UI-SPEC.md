## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, toolbar icon internal spacing |
| sm | 8px (`gap-2`, `p-2`) | Toolbar button gaps, dropdown item padding |
| md | 16px (`gap-4`, `p-4`) | Chart card internal padding, grid toolbar internal spacing |
| lg | 24px (`gap-6`, `p-6`) | Page padding, fullscreen modal internal padding |
| xl | 32px (`gap-8`) | Fullscreen modal header-to-content gap |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Chart hover toolbar icon buttons: 28px touch target (`size-7` = 28px) -- smaller than standard 32px to avoid crowding the chart card header
- Grid toolbar height: 36px (`h-9`) -- matches existing Input/Button small size pattern from `grid-toolbar.tsx`
- Auto-refresh dropdown trigger: `h-8 px-2` -- compact inline control that sits in the dashboard header without dominating

**Source:** CLAUDE.md (page padding `p-6`, section gaps `gap-6`, grid gaps `gap-4`), existing card padding patterns in `config-chart-grid.tsx`

## Interaction Contract

### Chart Hover Toolbar

| Interaction | Gesture | Response | Latency Target |
|-------------|---------|----------|----------------|
| Show toolbar | Mouse enters chart card | Toolbar fades in (top-right of CardHeader) | 0ms (instant show after 0ms delay) |
| Hide toolbar | Mouse leaves chart card | Toolbar fades out after 150ms delay | 150ms exit delay to prevent flicker |
| Open export menu | Click export icon in toolbar | DropdownMenu opens below the icon | <16ms |
| Export as PNG | Click "PNG Image" in export dropdown | Download starts, toast confirms "Chart exported as PNG" | <500ms (chart render to blob) |
| Export as SVG | Click "SVG Image" in export dropdown | Download starts, toast confirms "Chart exported as SVG" | <500ms |
| Export as CSV | Click "CSV Data" in export dropdown | Download starts, toast confirms "Data exported as CSV" | <100ms |
| Copy to clipboard | Click "Copy to Clipboard" in export dropdown | Tab-separated data copied, toast confirms "Data copied to clipboard" | <100ms |
| Open fullscreen | Click expand icon in toolbar | Dialog opens with chart at full viewport size | 200ms (Dialog animate-in) |
| Close fullscreen | Click X, press Escape, or click overlay | Dialog closes, return to dashboard | 200ms (Dialog animate-out) |
| Refresh chart | Click refresh icon in toolbar (per-chart) | Chart data re-fetched, spinner on icon during fetch | Standard fetch latency |

**Source:** CONTEXT.md D-01 (hover-reveal), D-08 (PNG), D-09 (SVG), D-10 (CSV), D-11 (clipboard), D-12 (WYSIWYG export)

### Export Behavior Details

| Export Type | Method | Filename Convention | Resolution | WYSIWYG |
|-------------|--------|--------------------|-----------:|---------|
| PNG (AG Charts) | `AgChartInstance.download('image/png')` | `{chartTitle}-{YYYY-MM-DD}.png` | 2x device pixel ratio (retina) | Yes -- includes cross-filter dimming |
| SVG (AG Charts) | `AgChartInstance.download('image/svg+xml')` | `{chartTitle}-{YYYY-MM-DD}.svg` | Vector (scalable) | Yes |
| PNG (ECharts) | `echartsInstance.getDataURL({ type: 'png', pixelRatio: 2 })` + manual `<a>` download | `{chartTitle}-{YYYY-MM-DD}.png` | 2x pixel ratio | Yes -- includes highlight/downplay state |
| SVG (ECharts) | `echartsInstance.getDataURL({ type: 'svg' })` + manual `<a>` download | `{chartTitle}-{YYYY-MM-DD}.svg` | Vector | Yes |
| CSV (chart data) | Build CSV string from `data.columns` + `data.data`, trigger download via `Blob` + `URL.createObjectURL` | `{chartTitle}-{YYYY-MM-DD}.csv` | N/A | N/A -- exports raw data |
| Clipboard | Build TSV string (tab-separated) from `data.columns` + `data.data`, `navigator.clipboard.writeText(tsv)` | N/A | N/A | N/A -- exports raw data |

Filename sanitization: Replace spaces with hyphens, remove special characters, lowercase. Example: `TLM Breaks by Region` becomes `tlm-breaks-by-region-2026-04-05.png`.

**Source:** CONTEXT.md D-08 through D-12, existing clipboard pattern in `explorer/query-results.tsx`

### Manual Refresh

| Interaction | Gesture | Response | Error Handling |
|-------------|---------|----------|----------------|
| Dashboard refresh | Click refresh button in `DashboardToolbar` | All `['data-source', *]` queries invalidated via `queryClient.invalidateQueries({ queryKey: ['data-source'] })`. Spinner replaces refresh icon. Previous data stays visible (`placeholderData: keepPreviousData`). | On failure: keep old data, toast "Refresh failed: {reason}. Showing cached data." with retry action in toast. |
| Per-chart refresh | Click refresh icon in chart hover toolbar | Single chart's query invalidated via `queryClient.invalidateQueries({ queryKey: ['data-source', dataSourceId] })`. Spinner on that chart's toolbar icon only. | Same error handling: keep old data, toast error. |

**Source:** CONTEXT.md D-15 (dashboard-level), D-16 (keep previous data), D-17 (error handling)

### Manual Refresh Visual States

| State | Dashboard Refresh Button | Chart Toolbar Refresh Icon | Chart Content |
|-------|-------------------------|---------------------------|---------------|
| Idle | `RefreshCw` icon, static | `RefreshCw` icon, static | Normal rendering |
| Refreshing | `RefreshCw` with `animate-spin`, button disabled | `RefreshCw` with `animate-spin` | Previous data visible, no skeleton, no blank. Subtle `opacity-75` on chart content during fetch. |
| Success | Icon stops spinning, no explicit success indicator | Icon stops spinning | New data renders (smooth transition via chart animation) |
| Failure | Icon stops spinning, toast notification | Icon stops spinning, toast notification | Old data remains. Toast: "Refresh failed: {reason}. Showing cached data." |

### Auto-Refresh

| Interaction | Gesture | Response |
|-------------|---------|----------|
| Open interval selector | Click auto-refresh dropdown in `DashboardToolbar` | DropdownMenu shows interval presets |
| Select interval | Click preset (Off, 1m, 5m, 10m, 30m) | Interval saved, countdown starts, active dot appears |
| View countdown | Glance at auto-refresh control | `{M}:{SS}` countdown text next to dropdown |
| Auto-refresh fires | Timer reaches 0 | Same behavior as manual refresh (invalidate all, keep previous data, spinner) |
| Disable auto-refresh | Select "Off" from dropdown | Timer stops, active dot disappears, countdown hidden |

## Chart Wrapper Ref API

Both chart wrappers must expose an imperative handle via `forwardRef` + `useImperativeHandle`.

### AgChartWrapper Ref

```typescript
interface AgChartRef {
  /** Download chart as image. type: 'image/png' | 'image/svg+xml' */
  download: (type: string, fileName: string) => void
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Chart toolbar visibility | `aria-hidden="true"` when toolbar is hidden (mouse not hovering). Toolbar icons gain focus via keyboard only when card is focused. |
| Export dropdown | Radix DropdownMenu handles `aria-expanded`, `aria-haspopup`, keyboard navigation (arrow keys, Enter, Escape) |
| Fullscreen dialog | Radix Dialog handles `aria-modal`, `role="dialog"`, focus trap, Escape key dismissal |
| Fullscreen dialog title | `DialogTitle` provides `aria-labelledby` for the dialog |
| Refresh button state | `aria-label="Refresh all dashboard data"` when idle. `aria-label="Refreshing..."` + `aria-busy="true"` when refreshing. |
| Auto-refresh countdown | `aria-live="off"` on countdown text (avoid constant screen reader announcements). Active state announced via `aria-label` on dropdown trigger: "Auto-refresh: 10 minutes, next refresh in {time}" |
| Grid toolbar buttons | All buttons have visible text labels (no icon-only buttons in grid toolbar) |
| Chart toolbar icon-only buttons | Each icon button has `aria-label`: "Export chart", "View fullscreen", "Refresh data" |
| Keyboard: chart card | Tab focuses card, then toolbar icons become focusable. Arrow keys navigate between toolbar icons. |
| Toast notifications | Sonner handles `role="status"` and `aria-live="polite"` |
