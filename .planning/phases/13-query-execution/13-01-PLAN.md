    **1. `normalize_columns(column_names: list[str]) -> list[str]`**
    Lowercase all column names. This handles QENG-06 (Oracle UPPERCASE normalization).
    ```python
    def normalize_columns(column_names: list[str]) -> list[str]:
        return [c.lower() for c in column_names]
    ```

    **2. `detect_column_type(db_type_name: str) -> tuple[str, bool]`**
    Map database type names from cursor.description to RecViz types. Return `(recviz_type, is_date)`.
    Handles QENG-05.

    Mapping rules:
    - Types containing `CHAR`, `VARCHAR`, `TEXT`, `CLOB`, `STRING`, `NCHAR`, `NVARCHAR` -> `("string", False)`
    - Types containing `NUMBER`, `INT`, `FLOAT`, `DECIMAL`, `NUMERIC`, `DOUBLE`, `REAL`, `MONEY` -> `("number", False)`
    - Types containing `DATE`, `TIMESTAMP`, `TIME` -> `("date", True)`
    - Default -> `("string", False)`
    - Match case-insensitively on the uppercase of the type name.

    **3. `build_result_response(cursor_columns, rows, max_rows: int = 10_000) -> dict`**
    Build the response dict matching the Superset contract. Parameters:
    - `cursor_columns`: list of tuples from `result.cursor.description` — each tuple is `(name, type_code, ...)`. OR list of `(name, type_name_str)` tuples for easier testing.
    - `rows`: list of tuples (raw DB rows).
    - `max_rows`: row limit.

    Implementation:
    ```python
    def build_result_response(
        column_descriptions: list[tuple],
        rows: list[tuple],
        max_rows: int = 10_000,
    ) -> dict:
        # Normalize column names (Oracle UPPERCASE -> lowercase)
        col_names = normalize_columns([desc[0] for desc in column_descriptions])
        # Detect types from description
        col_types = []
        for desc in column_descriptions:
            type_name = desc[1] if isinstance(desc[1], str) else getattr(desc[1], "__name__", str(desc[1]))
            recviz_type, is_date = detect_column_type(type_name)
            col_types.append({"column_name": col_names[col_types.__len__()], ...})
        # Actually, compute index properly:
        columns_meta = []
        for i, desc in enumerate(column_descriptions):
            type_name = desc[1] if isinstance(desc[1], str) else getattr(desc[1], "__name__", str(desc[1]))
            recviz_type, is_date = detect_column_type(type_name)
            columns_meta.append({
                "column_name": col_names[i],
                "name": col_names[i],
                "type": recviz_type,
                "is_date": is_date,
            })

        truncated = len(rows) > max_rows
        limited_rows = rows[:max_rows] if truncated else rows

        row_dicts = [dict(zip(col_names, row)) for row in limited_rows]

        return {
            "columns": columns_meta,
            "rows": row_dicts,
            "row_count": len(row_dicts),
            "truncated": truncated,
        }
    ```

    **4. `validate_read_only(sql: str) -> bool`**
    Return True if the SQL is a read-only statement (SELECT or WITH). Handles QENG-04.
    ```python
    import re
    _FORBIDDEN_PREFIXES = re.compile(
        r"^\s*(INSERT|UPDATE|DELETE|DROP|ALTER|CREATE|TRUNCATE|MERGE|GRANT|REVOKE)\b",
        re.IGNORECASE,
    )

    def validate_read_only(sql: str) -> bool:
        return not bool(_FORBIDDEN_PREFIXES.match(sql))
    ```

    **5. `wrap_with_pagination(sql: str, limit: int | None, offset: int = 0, dialect: str = "postgresql") -> str`**
    Wrap SQL with dialect-aware pagination. Handles DIAL-02.
    - If `limit` is None, return sql unchanged.
    - PostgreSQL: append `\nLIMIT {limit} OFFSET {offset}`
    - Oracle: wrap in subquery + `\nOFFSET {offset} ROWS FETCH FIRST {limit} ROWS ONLY`
