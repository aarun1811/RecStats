## Directory Purposes

**`frontend/src/routes/`:**
- Purpose: File-based routes consumed by TanStack Router; auto-compiled into `routeTree.gen.ts` by the Vite plugin
- Contains: `__root.tsx`, `_app.tsx` (authenticated layout), `index.tsx` (redirect), nested `_app/<domain>/` folders for each section, and `embed/dashboards/` for iframe-friendly dashboards
- Key files: `__root.tsx`, `_app.tsx`, `_app/dashboards/$dashboardId.tsx`, `_app/dashboards/$dashboardId.edit.tsx`, `embed/dashboards/$dashboardId.tsx`

**`frontend/src/components/ui/`:**
- Purpose: Shadcn/ui primitives ‚Äî owned code, not a dependency
- Contains: Tailwind-based wrappers around Radix primitives: `button.tsx`, `card.tsx`, `dialog.tsx`, `sheet.tsx`, `select.tsx`, `sidebar.tsx`, `resizable.tsx`, `sonner.tsx`, `empty.tsx`, `spinner.tsx`, `kbd.tsx`, `timeline.tsx`, etc.
- Rule: Never modify these files directly. Extend via composition in domain components.

**`frontend/src/components/layout/`:**
- Purpose: Application chrome
- Contains: `app-sidebar.tsx`, `header.tsx`, `nav-main.tsx`, `nav-user.tsx`, `search.tsx`, `command-palette.tsx`, `theme-provider.tsx`, `theme-switch.tsx`

**`frontend/src/components/dashboard/`:**
- Purpose: Config-driven runtime components that render a saved dashboard
- Contains: `dashboard-renderer.tsx` (entrypoint), `config-filter-bar.tsx`, `config-kpi-row.tsx`, `config-chart-grid.tsx`, `config-data-grid.tsx`, `dashboard-toolbar.tsx`, `drill-breadcrumb.tsx`, `drill-detail-grid.tsx`, `cross-filter-bar.tsx`, `chart-fullscreen-dialog.tsx`, `chart-toolbar.tsx`, `auto-refresh-control.tsx`, `share-link-button.tsx`, `dashboard-mini-map.tsx`, plus dashboard library cards/rows/list/toolbar.
- Rule: ACTIVE components are all `config-*.tsx`. Legacy components like `filter-bar.tsx`, `kpi-row.tsx`, `chart-grid.tsx` are DEAD CODE ‚Äî do not extend them. See CONCERNS.md.

**`frontend/src/components/builder/`:**
- Purpose: Dashboard builder ‚Äî the wizard that lets users compose dashboards visually
- Contains: `builder-page.tsx` (entrypoint), `builder-canvas.tsx` (RGL grid), `builder-toolbar.tsx`, `builder-panel.tsx`, `builder-panel-content.tsx`, `builder-filter-bar.tsx`, `add-content-menu.tsx`, dialogs for picking charts/KPIs/datasets (`chart-picker-dialog.tsx`, `kpi-picker-dialog.tsx`, `dataset-picker-dialog.tsx`), `filter-config-dialog.tsx`, `save-dashboard-dialog.tsx`, `delete-dashboard-dialog.tsx`, `drill-hierarchy-editor.tsx`, `filter-column-mapper.tsx`, `panel-config-popover.tsx`, `unsaved-changes-guard.tsx`, `builder-empty-state.tsx`

**`frontend/src/components/charts/`:**
- Purpose: Chart rendering and chart builder library
- Contains: `chart-factory.tsx` (routes to AG/ECharts), `ag-chart-wrapper.tsx`, `echart-wrapper.tsx`, `chart-type-icon.tsx`, `unsupported-chart-error.tsx`, `column-missing-error.tsx`, chart library/row/card/toolbar/detail-panel, `chart-builder.tsx`, `chart-builder-preview.tsx`, `delete-chart-dialog.tsx`
- Subdirectory: `builder/` ‚Äî wizard steps (`step-dataset.tsx`, `step-type.tsx`, `step-mapping.tsx`, `step-appearance.tsx`, `step-save.tsx`)

**`frontend/src/components/kpis/`:**
- Purpose: KPI library and KPI builder
- Contains: `kpi-library-list.tsx`, `kpi-library-card.tsx`, `kpi-library-row.tsx`, `kpi-library-toolbar.tsx`, `kpi-detail-panel.tsx`, `kpi-preview-card.tsx`, `kpi-builder.tsx`, `kpi-builder-preview.tsx`, `delete-kpi-dialog.tsx`
- Subdirectory: `builder/` ‚Äî wizard steps (`step-dataset.tsx`, `step-column.tsx`, `step-format.tsx`, `step-thresholds.tsx`, `step-trend.tsx`)

**`frontend/src/components/datasets/`:**
- Purpose: Dataset library and editor
- Contains: `dataset-list.tsx`, `dataset-list-toolbar.tsx`, `dataset-card.tsx`, `dataset-row.tsx`, `dataset-editor.tsx`, `column-metadata-grid.tsx`, `format-preset-select.tsx`, `dataset-sql-rerun-banner.tsx`, `delete-dataset-dialog.tsx`

- Contains: `error-boundary.tsx`, `error-panel.tsx`, `count-animation.tsx` (motion/react KPI counter), `page-transition.tsx` (motion/react route fade)

**`frontend/src/hooks/`:**
- Purpose: TanStack Query hooks and custom React hooks
- Contains: Query/mutation hooks per domain ‚Äî `use-managed-dashboards.ts`, `use-managed-charts.ts`, `use-managed-kpis.ts`, `use-managed-datasets.ts`, `use-data-source-query.ts`, `use-data-source-merge.ts`, `use-databases.ts`, `use-tables.ts`, `use-table-columns.ts`, `use-saved-views.ts`, `use-search.ts`, `use-sql-execute.ts`, `use-sql-history.ts`, `use-filter-options.ts`, `use-dashboard-kpis.ts`, `use-drill-detail.ts`, `use-drill-down.ts`, `use-cross-filter.ts`, `use-cross-filter-data.ts`, `use-auto-refresh.ts`, `use-builder-keyboard-shortcuts.ts`, `use-mobile.ts`

**`frontend/src/stores/`:**
- Purpose: Zustand stores ‚Äî client-only UI state
- Contains: `filter-store.ts` (global filters, applied snapshot, cross-filters), `drill-store.ts` (per-chart drill stacks), `builder-store.ts` (in-progress dashboard edits), `layout-history-store.ts` (undo/redo for RGL layouts)

**`frontend/src/lib/`:**
- Purpose: Cross-cutting utilities, API client, query client, domain helpers
- Contains: `api-client.ts`, `query-client.ts`, `utils.ts` (cn), `formatters.ts`, `chart-themes.ts`, `chart-export.ts`, `chart-compatibility.ts`, `cross-filter.ts`, `column-merge.ts`, `column-detection.ts`, `dashboard-url-state.ts`, `kpi-aggregator.ts`, `kpi-utils.ts`

**`frontend/src/types/`:**
- Purpose: Shared TypeScript types and Pydantic response shapes
- Contains: `dashboard-config.ts` (DashboardConfig, FilterConfig, KpiConfig, DashboardChartConfig, GridConfig, KpiResult, DataSourceQueryResponse), `dashboard.ts`, `chart.ts` (ChartConfig, ChartDataResponse, ChartWrapperProps, ChartRef), `filter.ts` (FilterValue, CrossFilter, DrillLevel), `builder.ts`, `formatting.ts`, `api.ts`, `database.ts`, `schema.ts`, `views.ts`, `managed-dashboard.ts`, `managed-chart.ts`, `managed-kpi.ts`, `managed-dataset.ts`

**`backend/app/api/`:**
- Purpose: FastAPI route handlers ‚Äî one router file per domain, all mounted by `router.py`
- Contains: `managed_dashboards.py`, `managed_charts.py`, `managed_kpis.py`, `managed_datasets.py`, `data_sources.py`, `databases.py`, `sql.py`, `search.py`, `views.py`
- Key files: `router.py` ‚Äî the aggregator every new domain router must be added to

**`backend/app/services/`:**
- Purpose: Business logic that is not tied to the HTTP layer
- Contains: `query_engine.py`, `engine_manager.py`, `connection_resolver.py`, `connection_status.py`, `config_store.py`, `merge_engine.py`, `encryption.py`, `uri_builder.py`, `query_utils.py`

**`backend/app/models/`:**
- Purpose: Pydantic v2 request/response models ‚Äî the HTTP-level shape
- Contains: `base.py` (CamelModel base class), per-domain models (`managed_dashboard.py`, `managed_chart.py`, `managed_kpi.py`, `managed_dataset.py`, `data_source_config.py`, `database.py`, `chart_data.py`, `filters.py`, `views.py`, `error.py`)
- Rule: All request/response models subclass `CamelModel` so wire format is camelCase while Python stays snake_case.

**`backend/app/db/`:**
- Purpose: Database engine, session factory, and ORM models for the RecViz metadata DB
- Contains: `engine.py` (sync engine, `session_factory`), `base.py` (DeclarativeBase), `types.py` (OracleJSON), `models/` with one file per table
- Key files: `db/models/dashboard.py`, `db/models/chart.py`, `db/models/kpi.py`, `db/models/dataset.py`, `db/models/data_source.py`, `db/models/connection.py`

**`backend/app/migrations/`:**
- Purpose: Alembic migration environment and versioned migrations
- Contains: `alembic.ini`, `env.py`, `versions/001_initial_schema.py` through `versions/007_dataset_database_id_to_string.py`
- Gotcha: Uses `recviz_alembic_version` table to avoid conflicts with Superset's default `alembic_version` table on the same PostgreSQL database.

**`backend/app/core/`:**
- Purpose: Cross-cutting framework code ‚Äî dependency injection, error sanitization
- Contains: `dependencies.py` (Annotated DI types), `errors.py` (`sanitize_detail`)

## Naming Conventions

**Files:**
- Frontend components: `kebab-case.tsx` (e.g. `dashboard-renderer.tsx`, `config-filter-bar.tsx`)
- Frontend hooks: `use-{name}.ts` (e.g. `use-managed-dashboards.ts`)
- Frontend stores: `{name}-store.ts` (e.g. `filter-store.ts`)
- Frontend types: `kebab-case.ts` in `types/` (e.g. `dashboard-config.ts`, `managed-chart.ts`)
- Frontend utilities: `kebab-case.ts` in `lib/` (e.g. `api-client.ts`, `cross-filter.ts`)
- Frontend unit tests: `{name}.test.ts(x)` co-located with the source file
- Python files: `snake_case.py` (e.g. `query_engine.py`, `managed_dashboards.py`)
- Migrations: `{order}_{description}.py` (e.g. `001_initial_schema.py`, `007_dataset_database_id_to_string.py`)

**Routes (TanStack Router file-based):**
- List/index pages: `index.tsx` (e.g. `routes/_app/dashboards/index.tsx`)
- Detail pages: `$paramName.tsx` (e.g. `routes/_app/dashboards/$dashboardId.tsx`)
- Nested child routes via dot-notation: `$dashboardId.edit.tsx` is a child of `$dashboardId.tsx`
- Layout routes prefixed with `_`: `_app.tsx` is a layout, `_app/` contains its children
- Authenticated-only routes nest under `_app/`; embed routes nest under `embed/`

**Identifiers:**
- TypeScript: camelCase for variables/functions, PascalCase for components and types/interfaces, UPPER_SNAKE_CASE for constants (e.g. `DATA_KEYS`, `ECHART_TYPES`, `SUPPORTED_AG_TYPES`)
- Python: snake_case for variables/functions, PascalCase for classes, UPPER_SNAKE_CASE for module constants (e.g. `DEFAULT_MAX_ROWS`, `HEALTH_CHECK_SQL`)
- Private Python helpers: leading underscore (`_to_response`, `_build_sql`, `_resolve_database`)

**Component naming:**
- Props interface named `{ComponentName}Props` and defined directly above the component
- Components use named function declarations (`export function DashboardRenderer()`), not arrow functions
- Route components export `const Route = createFileRoute(...)({ component: LocalComponent })`

**Database tables:**
- All RecViz-managed tables prefixed `recviz_`: `recviz_dashboards`, `recviz_charts`, `recviz_kpis`, `recviz_datasets`, `recviz_data_sources`, `recviz_connections`
- Alembic version table: `recviz_alembic_version` (override to avoid Superset conflict)

**API routes:**
- All under `/api/` prefix
- Managed entities under `/api/{entity}/managed` (e.g. `/api/dashboards/managed`, `/api/charts/managed`, `/api/kpis/managed`, `/api/datasets/managed`)
- Data source queries under `/api/data-sources/{id}/{action}`
- SQL explorer under `/api/sql/*`
- Search under `/api/search`
- Database/connection CRUD under `/api/databases/*`

**TanStack Query keys:**
- `['entity', identifier, filters]` shape
- Examples: `['managed-dashboards']`, `['managed-dashboard', id]`, `['data-source', dataSourceId, filters]`, `['dashboard-kpis', kpis, filters]`

## Where to Add New Code

**New page/route:**
- Create file at `frontend/src/routes/_app/<section>/<name>.tsx` (or `index.tsx` / `$param.tsx`)
- TanStack Router Vite plugin regenerates `routeTree.gen.ts` automatically
- Export `const Route = createFileRoute(...)({ component: LocalComponent })`

**New dashboard runtime component:**
- Add under `frontend/src/components/dashboard/` with prefix `config-` if it is part of the config-driven renderer
- Example home: `frontend/src/components/dashboard/config-<name>.tsx`

**New chart type:**
- Add to `frontend/src/components/charts/chart-factory.tsx` ‚Äî update `ECHART_TYPES` or `SUPPORTED_AG_TYPES`
- Implement inside `AgChartWrapper` (`ag-chart-wrapper.tsx`) or `EChartWrapper` (`echart-wrapper.tsx`)
- Update `ChartType` union in `frontend/src/types/chart.ts`
- Add icon mapping in `frontend/src/components/charts/chart-type-icon.tsx`
- Add to chart builder wizard in `frontend/src/components/charts/builder/step-type.tsx`

**New builder wizard step (chart/KPI/dataset):**
- Chart: `frontend/src/components/charts/builder/step-<name>.tsx`
- KPI: `frontend/src/components/kpis/builder/step-<name>.tsx`
- Wire into the parent builder (`chart-builder.tsx`, `kpi-builder.tsx`)

**New Zustand store:**
- File: `frontend/src/stores/<name>-store.ts`
- Define `interface {Name}Store` directly above `create<{Name}Store>()`
- Use selectors at the call site to avoid unnecessary re-renders

**New TanStack Query hook:**
- File: `frontend/src/hooks/use-<name>.ts`
- Wrap `useQuery` / `useMutation` and return the result object directly
- Use query key convention `['entity', id, filters]`
- Invalidate related keys in mutation `onSuccess`

**New shared TypeScript type:**
- File: `frontend/src/types/<name>.ts`
- Import via `@/types/<name>`; use `import type` (enforced by `verbatimModuleSyntax`)

**New API endpoint (backend):**
- Create or extend a router file in `backend/app/api/<domain>.py`
- Use `def` (not `async def`) if it touches the DB (threadpool model)
- Request/response models in `backend/app/models/<domain>.py`, subclass `CamelModel`
- Register the router in `backend/app/api/router.py` via `api_router.include_router(...)`
- Use `DbSessionDep`, `ConfigStoreDep`, `QueryEngineDep`, `EngineManagerDep`, `ConnectionResolverDep`, `ResolvedDataSourceDep` as appropriate

**New service:**
- File: `backend/app/services/<name>.py`
- If it is long-lived: instantiate in `backend/app/main.py` lifespan and store on `app.state`
- Expose a `DepType = Annotated[ServiceType, Depends(factory)]` in `backend/app/core/dependencies.py`

**New DB model / table:**
- File: `backend/app/db/models/<name>.py`
- Subclass `Base`, use `Mapped[T]` + `mapped_column()`, prefix table name with `recviz_`
- Use `OracleJSON()` from `backend/app/db/types.py` for JSON columns
- Create an Alembic migration under `backend/app/migrations/versions/<nnn>_<desc>.py`
- Import the new model in any module that needs Alembic autogenerate to see it (usually `db/models/__init__.py`)

**New SQLAlchemy ORM import for an existing table:**
- Add to `backend/app/db/models/__init__.py` if autogenerate needs to see it

**New migration:**
- Command: `alembic upgrade head` applies; write files manually or via `alembic revision -m "..."`
- File: `backend/app/migrations/versions/<next_number>_<desc>.py`
- Remember the version table is `recviz_alembic_version`

**Shared utility:**
- Frontend: `frontend/src/lib/<name>.ts`
- Backend: `backend/app/services/query_utils.py` or a new module in `backend/app/services/`
