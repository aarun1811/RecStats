## Notes for Subsequent Agents

- The Shadcn `SidebarProvider` manages its own open/closed state via cookie + context. The `useSidebarStore` from scaffolding is available but the Shadcn sidebar has its own built-in state management (including `Cmd+B` toggle shortcut). Both can coexist if needed.
- The Sonner `Toaster` was updated to use `useThemeStore` instead of `next-themes` — the `next-themes` package can be safely removed from dependencies if not used elsewhere.
- Route tree (`routeTree.gen.ts`) is auto-generated by the `@tanstack/router-plugin` Vite plugin on every build/dev start. Do not edit it manually.
- All route page components are placeholders — Agents 03 (Dashboard) and 06 (Explorer) should replace the placeholder content.
- The `CommandPalette` has a "Dashboards" group ready for dynamic dashboard items — Agent 03 can populate this.
